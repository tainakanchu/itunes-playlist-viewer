# 選曲ワークスペース (Crateforge)

このディレクトリは **DJ 選曲の作業場**。ここで Claude Code を開き、`dj-curator` プラグインの
`/dj-curator:build-set` で叩き台を作る。初回の方針づくりは `/dj-curator:init-workspace` が手伝う。
Crateforge アプリの内蔵 API（既定 `http://127.0.0.1:8787`）を有効にしておくこと。

> これは **持続的なプロファイル**（DJ の色・基準・運用則）。各セット固有の文脈（スロット / 会場 / 尺 / 今日のムード）は
> build-set が都度ヒアリングする。ここには「よくやる現場の既定値」を書く。空欄は記入例を残して後から育てればよい。

---

## 選曲の基本方針 (ここを自分用に書き換える)

### アイデンティティ / 音楽の色
- 例: Deep House / Melodic Techno / Disco Re-edit を軸に。地続きなら越境 OK、露骨なジャンルジャンプは避ける。
- 例: 攻めすぎない大人のフロア。歌ものは要所に。
- シグネチャー曲（必ず候補に入れたい固定枠）: 例「<曲> を世界観の基準点に」。

### テンポ・キー・エナジー
- ホームテンポ 120–126。夕方は 90 台から漸増（必要ならハーフ/ダブルタイムで接続）。
- キー: 隣同士は Camelot 0・±1・A↔B が好み。+2 の energy boost は山場で時々。
- エナジー: ±1 ずつ動かす。ピークで energy 高めを 1〜2 曲、谷・箸休めも作る（全部バンガーにしない）。

### 母集団・品質・新鮮さ
- 既定の母集団は **★3 以上**（API では `ratingMin=60`）。ニッチ企画では絞らず母数優先。
- ★ の自己定義: 例 ★5＝実証済みでフロア対応 / ★3＝試したい / ★1＝候補保留。
- 年代: 例 2015 以降を中心に。音質が古すぎるものは避ける。
- 認知度バランス: クラウドプリーザー : ディグった深い曲 ＝ 7 : 3 くらい。
- 解析値（BPM/Key/Energy）はメタデータ主体の補助。未解析も候補に入れる。

### 声・言語
- vocal / インスト比率: 例 序盤は歌もの少なめ、ピークでアンセム 1〜2 曲。
- 言語・地域の軸: 例「台湾系。台語 / 華語 / 原住民語を区別」。
- **言語などタグに無い属性の捕捉レシピ**（再利用資産。気づいたら追記する）:
  - 代表アーティストを名指し検索（`q=<アーティスト>`）。
  - 言語の特有語でスイープ（例 台語: 毋 / 袂 / 媠 / 查某 / 鬥陣 / 暗暝 …）。
  - `GET /api/genres` で関連 token を洗い `genre=<token>` で横断。
- 歌詞テーマ・露骨さ: 例 普段は不問（結婚式/企業では NG リストを確認）。

### 避けたい / 重複回避
- 避けたい: 露骨な EDM ビッグルーム、obvious 過ぎる大曲の直球、音質が悪いもの。
- 直近セット重複回避: 過去 3 ヶ月の `sets/` で使った曲はなるべく避ける（play count も参照）。

### よくやる現場の既定値（セットごとに build-set が上書き）
- スロット: 例 ピーク前後が多い（warm-up ならピーク級を温存し energy 低めから）。
- 会場・イベント: 例 クラブ中心 / たまに配信。
- 尺・曲数感: 例 60–90 分 ≒ 15–25 曲（候補は必要量の約 2 倍出す）。

## 運用メモ
- 叩き台のプレイリストは Crateforge アプリに作られる。**曲順はアプリ上で詰める**。
- プレイリスト命名規則: 例「<コンセプト> (叩き台)」。
- セットの記録は `sets/<日付>-<イベント>.md` に残す（重複回避と振り返り用）。
- メタデータに無い属性は `comments` に `#tag`（例 `#台語` `#inst`）を付けると `q=#台語` で検索できる。
- フライヤー画像やテーマメモは `inputs/` に置くと投げやすい。
- 別ポートで動かしている場合は環境変数 `CRATEFORGE_API` を設定。
