Crateforge 以 **純 Rust 的 DSP** 進行 DJ 取向的音訊解析。
不依賴外部工具，即可推估 **BPM / Key (Camelot) / Energy / 響度** 與相似度向量。

> 截圖稍後補上

## 會解析哪些項目

| 指標 | 內容 |
|---|---|
| **BPM** | 自動偵測節奏。掌握容易銜接的曲目。 |
| **Key (Camelot)** | 為了和聲混音，以 Camelot 表記（例 `8A`）顯示調性。 |
| **Energy** | 將曲目的氣勢數值化。可設計整套 set 的流動。 |
| **響度** | 音量等級。用於實際波形的繪製與正規化。 |
| **相似度向量** | 用來牽引出氛圍相近曲目的內部表示。 |

解析結果中的 **Key / Energy** 可作為曲目表格的欄顯示，也會顯示在 Now Playing。

## 何時會進行解析

解析只會在背景對「常用的曲目」進行。由以下任一動作觸發。

- **播放** 曲目時
- 右鍵 →執行 **「Analyze」** 時

進度會顯示在工具列。並非一次解析音樂庫全部曲目，而是聚焦在實際會用到的曲目進行解析，做到不浪費。

## 相似度選曲（Similar）

在右側欄的 **Similar** 分頁，會根據解析值提議「下一手」。

- 右鍵點選曲目 →「Find similar」即可顯示 **Camelot 鍵相容 + 節奏相近** 的候選。
- 以 **Harmonic 切換鈕** 可進一步篩選出和聲相合的曲目。
- 結果可 **加入 Crate**（或雙擊播放）（若全部都已加入，會以 "All added" 停用）。

## 搜尋篩選語法

解析值也能在搜尋中篩選（與文字搜尋以 AND 結合）。

```text
bpm:120-128
key:8A
energy:60-100
```

## 與智慧型播放清單、AI 選曲的整合

- 可在[智慧型播放清單](../smart-playlists/)的條件中使用 BPM / Key / Energy。
- [內建 API 伺服器](../api-server/)也會回傳解析結果與相似曲，因此 dj-curator 等 AI 選曲能把解析值當作加分來運用。

:::note
選曲的基本方針是 **以中繼資料為主**（評等 / 類型 / 年代），
而 BPM / Key / Energy 則被視為「有就用的加分」。
:::
